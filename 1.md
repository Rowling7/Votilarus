导航页项目需求文档（最终版）

# 1. 项目概述
基于 Node.js + 原生前端 + SQLite3 的个人导航页，支持快捷方式图标与小组件自由布局、分类横向滚动切换、底部 Dock 停靠栏（macOS 风格）、搜索引擎快捷跳转、多设备响应式及全局样式隔离。

# 2. 技术栈
- **后端**：Node.js + Express
- **数据库**：SQLite3
- **前端**：原生 HTML、CSS、JavaScript（无框架）
- **组件化**：Web Components（Shadow DOM），确保样式隔离与高复用性
- **拖拽**：HTML5 Drag and Drop API
- **代码组织**：模块化 ES6 导入导出，Manager 模式管理业务逻辑

# 3. 响应式布局

## 3.1 桌面端（≥1024px）
- **左侧悬浮侧栏**：宽度 6%（可配置），整体垂直居中，包含头像和分类列表
- **右侧内容区**：占据剩余空间，顶部为搜索框，下方为分类面板网格布局
- **底部 Dock 栏**：固定在底部中央（可配置位置：左/右/底），macOS 风格鱼眼效果

## 3.2 平板端（768px–1023px）
- 侧栏宽度适当调整
- 网格列数根据屏幕宽度动态计算（约 8-10 列）
- Dock 栏保持显示

## 3.3 移动端（<768px）
- **左侧侧栏隐藏**：分类切换通过底部 Tab 栏实现
- **Dock 栏隐藏**：不显示
- **网格自适应**：根据屏幕宽度动态计算列数（约 4-5 列）
- **内容区全宽**：右侧区域占满屏幕

## 3.4 滚动行为
- 中间内容区支持横向滑动切换分类
- 桌面端可按住 Shift + 滚轮横向滚动
- 移动端自然滑动
- **全局隐藏滚动条**（使用 CSS `::-webkit-scrollbar { display: none; }`）

# 4. 核心功能

## 4.1 左侧悬浮侧栏

### 设计要求
- **位置**：悬浮在屏幕左侧，整体垂直居中
- **宽度**：默认 6%，可在设置中调整
- **背景**：半透明毛玻璃效果
- **内容**：
  - 第一个固定位置：用户头像（点击弹出设置 Modal）
  - 其余为分类列表（从 A70 表取值，单列多行排列）
- **交互**：点击分类后，右侧内容区平滑滚动至对应分类面板

### 数据库关联
- 分类数据从 `A70` 表获取
- 使用 `uuid`（INTEGER 类型）作为分类 ID

## 4.2 头像 & 设置 Modal

### 头像
- 固定在侧栏第一个位置
- 点击弹出设置 Modal

### 设置项分类

#### 1. 基础设置
- **行数**：5-9，默认 5
- **列数**：13-20，默认 13（PC 端最低值）
- **侧栏宽度**：百分比，默认 6%

#### 2. 外观主题
- **主题模式**：dark/light/auto
- **主题色**：颜色选择器
- **背景图片 URL**：主区域背景图
- **背景模糊度**：0-20，默认 5
- **背景透明度**：0-1，默认 0.8
- **遮罩层颜色**：默认 #000000
- **遮罩层透明度**：0-1，默认 0.3

#### 3. 图标样式
- **图标圆角**：rem 单位，默认 0.5
- **图标阴影**：启用/禁用
- **图标悬停效果**：scale/glow/none
- **显示图标标题**：启用/禁用
- **标题位置**：bottom/top/floating
- **标题字体大小**：px 单位，默认 12px
- **标题字体颜色**：颜色选择器
- **标题最大长度**：字符数，默认 8（超过显示 ...）
- **气泡提示延迟**：ms 单位，默认 300ms

#### 4. Dock 设置
- **Dock 位置**：left/right/bottom，默认 bottom
- **Dock 最大图标数量**：默认 10
- **Dock 背景模糊度**：px 单位，默认 10px
- **Dock 背景透明度**：0-1，默认 0.3
- **鱼眼放大倍数**：默认 1.5
- **鱼眼影响范围**：左右各几个图标，默认 2

#### 5. 搜索设置
- **默认搜索引擎**：baidu/bing/google
- **搜索框位置**：center/left/right
- **搜索框样式**：rounded/square

#### 6. 交互行为
- **滚动动画速度**：ms 单位，默认 300ms
- **拖拽灵敏度**：px 单位，默认 5px
- **启用右键菜单**：启用/禁用

#### 7. 个人信息
- **头像 URL**：用户头像地址
- **用户名**：显示名称
- **个人简介**：简短介绍

#### 8. 高级设置
- **恢复默认设置**：重置所有配置
- **导出配置**：导出 JSON 文件
- **导入配置**：从 JSON 文件导入

### 数据持久化
- 所有设置存储在 `stettings` 表中
- 单用户设计，不考虑多账号

## 4.3 搜索框

### 位置与样式
- 位于右侧内容区顶部固定
- 不受侧栏影响
- 旁边显示当前搜索引擎图标
- 点击图标可切换引擎（百度/必应/谷歌）

### 功能
- 无本地过滤功能，仅提供搜索引擎跳转
- 输入关键词后按回车，在新标签页打开搜索结果
- **搜索引擎 URL**：
  - 百度：`https://www.baidu.com/s?wd=`
  - 必应：`https://www.bing.com/search?q=`
  - 谷歌：`https://www.google.com/search?q=`

## 4.4 右侧内容区（网格布局系统）

### 网格配置
- **PC 端**：最低 13 列 × 5 行，最大 20 列 × 9 行
- **移动端/平板端**：根据屏幕宽度动态计算列数
- **单元格大小**：基于 `cell_base_size`（默认 4rem）和 `cell_gap`（默认 1rem）
- **行列数调整**：可在设置中修改，已存在图标坐标自动按比例调整

### 分类面板
- 所有分类面板横向排列
- 点击左侧侧栏分类或横向滚动切换
- **不需要无缝循环**

### 图标/组件布局
- **定位方式**：绝对定位（position: absolute），坐标存储在数据库
- **拖拽移动**：拖拽结束后即时更新坐标并保存到数据库
- **尺寸规格**：1×1、1×2、1×3、1×4、2×1、2×2、2×3、2×4（默认 1×1）
- **图标与小组件共用同一套尺寸逻辑**

### 图标显示规则
- **有 bgimage**：直接显示 base64 图片作为背景图
- **无 bgimage**：显示首字母大写加粗（中文取第一个汉字，英文取第一个字母）
- **name 文字**：
  - 显示在图标下方
  - 超出最大长度自动省略（...）
  - hover 时气泡显示完整名称
  - 字体大小、颜色可在设置中调整

### 右键菜单
- **主区域图标右键**：
  - "设置大小" → 选择尺寸（1×1, 1×2, 1×3, 1×4, 2×1, 2×2, 2×3, 2×4）
  - "删除" → 确认后删除
- **空白区域右键**：
  - "添加图标"
  - "添加小组件"
  - "刷新"

## 4.5 底部 Dock 栏（macOS 风格）

### 样式设计
- **位置**：固定在底部中央（可通过设置切换 left/right/bottom）
- **背景**：毛玻璃效果（backdrop-filter: blur(10px)），透明度 0.3，适配暗黑模式
- **图标**：仅允许 1×1 大小的快捷方式图标
- **间距**：1rem
- **数量上限**：由设置控制（默认 10）

### 鱼眼效果
- **放大倍数**：默认 1.5（可在设置中调整）
- **影响范围**：hover 图标及其左右各两个图标（共 5 个）
- **实现方式**：CSS transform: scale() + transition
- **无弹跳动画**

### 交互规则
- **添加**：从主区域拖拽图标到 Dock → 复制引用（主区域图标保留）
- **删除**：在 Dock 图标上右键 → 右上角浮现红色 ❌ → 点击删除
- **Dock 内部拖拽**：调整图标顺序
- **从 Dock 拖拽到主区域**：**不允许**

### 移动端表现
- Dock 栏隐藏
- 使用底部 Tab 栏显示分类

# 5. 数据库设计

## 5.1 表结构说明

### A70 表（分类表）
- **uuid**：INTEGER 类型，主键，分类 ID
- **id**：INTEGER 类型
- **aindex**：TEXT 类型，分类索引名
- **name**：TEXT 类型，分类显示名称
- **crtDate**：DATE 类型，创建时间
- **upDate**：DATE 类型，更新时间
- **delDate**：DATE 类型，删除时间
- **isdel**：TEXT 类型，软删除标记（'0'=未删除，'1'=已删除）

### A7001 表（图标/项目表）
- **uuid**：TEXT 类型，主键，图标唯一标识
- **id**：INTEGER 类型
- **a70Id**：INTEGER 类型，外键，指向 A70.uuid
- **a70name**：TEXT 类型，分类名称
- **name**：TEXT 类型，图标显示名称
- **target**：TEXT 类型，链接地址
- **bgimage**：BLOB 类型，背景图片（base64）
- **crtDatetime**：DATE 类型，创建时间
- **delDatetime**：DATE 类型，删除时间
- **upDatetime**：DATE 类型，更新时间
- **isdel**：TEXT 类型，软删除标记

### item_layouts 表（布局信息表）
- **id**：INTEGER 类型，主键，自增
- **item_uuid**：TEXT 类型，外键，指向 A7001.uuid
- **category_id**：INTEGER 类型，外键，指向 A70.uuid
- **pos_x**：REAL 类型，X 坐标
- **pos_y**：REAL 类型，Y 坐标
- **width**：INTEGER 类型，宽度（格数）
- **height**：INTEGER 类型，高度（格数）
- **sort_order**：INTEGER 类型，排序顺序
- **is_active**：TEXT 类型，激活状态
- **created_at**：DATETIME 类型，创建时间
- **updated_at**：DATETIME 类型，更新时间

### dock_items 表（Dock 栏项表）
- **id**：INTEGER 类型，主键，自增
- **item_uuid**：TEXT 类型，外键，指向 A7001.uuid
- **sort_order**：INTEGER 类型，排序顺序
- **created_at**：DATETIME 类型，创建时间

### stettings 表（设置表）
- **id**：INTEGER 类型，主键
- **key**：TEXT 类型，设置键名
- **value**：TEXT 类型，设置值
- **type**：TEXT 类型，设置类型（personal/global）
- **notes**：TEXT 类型，备注说明
- **isdisplay**：TEXT 类型，是否显示
- **isdel**：TEXT 类型，软删除标记

## 5.2 关键关联关系
- `A7001.a70Id` → `A70.uuid`（INTEGER）
- `item_layouts.item_uuid` → `A7001.uuid`（TEXT）
- `item_layouts.category_id` → `A70.uuid`（INTEGER）
- `dock_items.item_uuid` → `A7001.uuid`（TEXT）

## 5.3 数据持久化
- 使用 SQLite3 存储所有数据
- 所有配置项持久化到 `stettings` 表
- 图标位置、大小等布局信息存储在 `item_layouts` 表

# 9. 技术实现要点

## 9.1 网格系统动态计算
```javascript
// 移动端/平板端动态计算列数
if (width < 768px) {
  columns = Math.floor(width / (cell_base_size * 16 + cell_gap * 16)) // 约 4-5 列
} else if (width < 1024px) {
  columns = Math.floor(width / (cell_base_size * 16 + cell_gap * 16)) // 约 8-10 列
} else {
  columns = userSetting.columns // 用户设置值（13-20）
}
```

## 9.2 坐标调整算法
当用户调整行列数时，已存在图标坐标按比例缩放：
```javascript
new_pos_x = old_pos_x * (new_columns / old_columns)
new_pos_y = old_pos_y * (new_rows / old_rows)
```

## 9.3 鱼眼效果实现
使用 JavaScript 计算每个图标的缩放比例（基于鼠标距离）：
```javascript
// hover 时，根据鼠标距离计算缩放比例
```
