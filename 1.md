导航页项目需求文档（最终精简版）
1. 项目概述
基于 Node.js + 原生前端 + SQLite3 的个人导航页，支持快捷方式图标与小组件自由布局、分类横向循环滚动切换、底部 Dock 停靠栏、搜索引擎快捷跳转、多设备响应式及全局样式隔离。

2. 技术栈
后端：Node.js + Express

数据库：SQLite3（表结构待定，按实际需求设计）

前端：原生 HTML、CSS、JavaScript（无框架）

样式方案：Tailwind CSS

组件复用与隔离：所有图标和小组件均封装为 Web Components（Shadow DOM），确保样式隔离与高复用性

拖拽：HTML5 Drag and Drop API

3. 响应式布局
桌面端（≥1024px）：左侧固定导航（宽度6%），右侧内容区（94%）

平板端（768px–1023px）：适当调整比例或转为顶部导航

移动端（<768px）：左侧导航转为底部 Tab 栏或汉堡菜单抽屉，右侧全宽；底部 Dock 栏隐藏至汉堡菜单内

中间内容区始终支持横向滑动切换分类，桌面端可按住 Shift + 滚轮 横向滚动，移动端自然滑动

4. 核心功能
4.1 左侧悬浮导航栏
竖向排列，第一个为头像，点击弹出设置 Modal

其余为分类列表，点击后右侧中间区域横向滚动至对应分类面板

导航栏始终悬浮（position: fixed），背景可半透明

4.2 头像 & 设置 Modal
点击头像弹出模态框，用于修改以下个性化设置（数据持久化到数据库）：

头像 URL

主题色

主区域背景图片及模糊度

默认搜索引擎（百度、必应、谷歌）

Dock 栏最大图标数量

分类的增删改排序

单用户设计，不考虑多账号

4.3 搜索框
位于右侧顶部固定

无本地过滤功能，仅提供搜索引擎跳转

输入关键词后按回车，使用预设的搜索引擎在新标签页打开搜索结果

百度：https://www.baidu.com/s?wd=

必应：https://www.bing.com/search?q=

谷歌：https://www.google.com/search?q=

搜索框旁边可显示当前搜索引擎图标，点击可切换引擎

4.4 右侧中间内容区（横向循环滚动分类面板）
分类切换：

左侧导航点击 → 平滑滚动至对应面板

Shift + 鼠标滚轮 或触控板横向滑动 → 左右循环滚动（最后一个面板后回到第一个，反之亦然）

移动端原生横向滑动支持循环

布局方式：所有分类面板横向排列，超出区域可滚动，通过 JS 实现循环效果

每个分类面板内部：

采用自由定位（记忆坐标），图标/组件使用绝对定位（position: absolute），位置坐标存储在数据库

拖拽移动后即时更新坐标并保存

组件大小可选宽高组合：1×1、1×2、1×3、1×4、2×1、2×2、2×3、2×4（默认 1×1），基础单元格 + 1rem 间距

右键菜单（主区域）：

“设置大小” → 选择尺寸

“删除” → 确认后删除

小组件：初期均为占位样式，预留功能扩展接口

4.5 底部 Dock 栏
固定在右侧区域底部，毛玻璃/半透明深色风格

仅允许 1×1 大小的快捷方式图标，间距 1rem，数量上限由设置控制（默认10）

添加：从中间区域拖拽图标到 Dock → 复制引用（主区域图标保留）

删除：在 Dock 图标上右键 → 右上角浮现红色 ❌ → 点击删除

移动端表现：Dock 栏不固定显示，隐藏在汉堡菜单抽屉中

4.6 Web Components 封装
图标：<nav-icon>，小组件：<nav-widget>，均通过 Shadow DOM 隔离样式

外部通过属性（如 title, url, size）进行配置

组件支持拖拽事件和自定义事件（如 delete-request）与主应用通信

4.7 搜索功能明细（仅跳转引擎）
无论是否匹配到本地内容，一律执行搜索引擎跳转

按下回车时自动调用 window.open(url + 编码后的搜索词, '_blank')

5. 数据持久化
使用 SQLite3 存储所有数据，具体表结构根据实际开发需求进行设计，至少涵盖分类、图标/组件（含位置、大小）、Dock项、用户设置等

6. 后端 API 草案
方法	路径	说明
GET	/api/categories	获取所有分类
POST	/api/categories	新增分类
PUT	/api/categories/:id	修改分类
DELETE	/api/categories/:id	删除分类
GET	/api/items?category_id=	获取分类下所有图标/组件
POST	/api/items	新建图标/组件
PUT	/api/items/:id	更新图标/组件（位置、大小等）
DELETE	/api/items/:id	删除图标/组件
GET	/api/dock	获取 Dock 栏所有项
POST	/api/dock	添加图标到 Dock（传入item_id）
DELETE	/api/dock/:id	从 Dock 删除项
GET	/api/settings	获取所有设置
PUT	/api/settings	批量更新设置
7. 关键交互流程简述
新增图标 → 调用接口 → 创建 <nav-icon> → 放置到分类面板

拖拽到 Dock → 调用接口复制 → 添加至 Dock 区域

移动图标位置 → 拖拽结束调用接口更新坐标

删除主区域项 → 右键删除 → 调用删除接口 → 移除 DOM

删除 Dock 项 → 右键浮现 ❌ → 点击确认 → 删除接口 → 移除 DOM
